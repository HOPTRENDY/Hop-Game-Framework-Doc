üé¨ AnimNotify Setup (CRITICAL)

‚ö†Ô∏è REQUIRED: Every combo animation montage MUST have HopComboGraphANS_ComboWindow AnimNotifyState configured, or combos will not chain properly!

Adding AnimNotifyState to Montages

Required
  1. Open your animation montage in the editor
  2. Find the Notifies track timeline
  3. Right-click on the timeline ‚Üí Add Notify State ‚Üí HopComboGraphANS_ComboWindow
  4. Position the notify state bar where you want the combo window
  5. Resize the bar to set window duration
  6. Save the montage
Recommended Timing 
Light Attack: 0.3s - 0.8s (mid animation)
Heavy Attack: 0.5s - 1.2s (later in animation)
Finisher: 0.8s - 1.5s (near end)
üí° Best Practice: Place combo windows in the middle-to-end of animations for smooth transitions. Avoid placing at the very start (0.0s) or very end.

How AnimNotify Events Work

Info

Event Flow:

  1. NotifyBegin fires ‚Üí Sends HopComboGraph.ComboBegin GameplayEvent
  2. Ability task receives event ‚Üí Opens combo window (bComboWindowOpened = true)
  3. Player presses input ‚Üí Transition is queued (not executed immediately)
  4. NotifyEnd fires ‚Üí Sends HopComboGraph.ComboEnd GameplayEvent
  5. Ability task receives event ‚Üí Closes window and executes queued transition
  6. New montage plays ‚Üí Repeat from step 1
‚úÖ Queued Transition Pattern: Inputs are queued during the window and executed when the window closes. This prevents animation interruption and ensures smooth transitions.

üíª C++ Usage

Creating ComboGraph Ability

C++
MyComboAbility.h 
UCLASS()
class UGA_MyCombo : public UGameplayAbility
{
    GENERATED_BODY()

public:
    UPROPERTY(EditDefaultsOnly, Category = "Combo")
    TObjectPtr<UHopComboGraph> ComboGraph;

    UPROPERTY(EditDefaultsOnly, Category = "Combo")
    TObjectPtr<UInputAction> InputAction;

protected:
    virtual void ActivateAbility(const FGameplayAbilitySpecHandle Handle,
        const FGameplayAbilityActorInfo* ActorInfo,
        const FGameplayAbilityActivationInfo ActivationInfo,
        const FGameplayEventData* TriggerEventData) override;
};
MyComboAbility.cpp 
void UGA_MyCombo::ActivateAbility(const FGameplayAbilitySpecHandle Handle,
    const FGameplayAbilityActorInfo* ActorInfo,
    const FGameplayAbilityActivationInfo ActivationInfo,
    const FGameplayEventData* TriggerEventData)
{
    if (!CommitAbility(Handle, ActorInfo, ActivationInfo))
    {
        EndAbility(Handle, ActorInfo, ActivationInfo, true, false);
        return;
    }

    // Create and activate combo graph task
    UHopComboGraphAbilityTask* ComboTask = UHopComboGraphAbilityTask::CreateTask(
        this, ComboGraph, InputAction);

    ComboTask->OnCompleted.AddDynamic(this, &UGA_MyCombo::OnComboCompleted);
    ComboTask->OnInterrupted.AddDynamic(this, &UGA_MyCombo::OnComboInterrupted);
    ComboTask->OnStateMachineTransition.AddDynamic(this, &UGA_MyCombo::OnComboTransition);

    ComboTask->ReadyForActivation();
}

Handling Input in C++

// In your character or component
void AMyCharacter::SetupPlayerInputComponent(UInputComponent* PlayerInputComponent)
{

    if (EnhancedInput && LightAttackAction)

    }
}

üîß Troubleshooting

Combo Not Chaining (Stuck on First Node)

Common Issue

Symptoms:

  • Only first animation plays
  • Pressing input during combo has no effect
  • Combo never advances to second node

Solution Checklist:

  1. ‚úÖ Verify AnimNotifyState is added to ALL montages
    • Open each montage
    • Look for green bar labeled "HopComboGraphANS_ComboWindow"
    • If missing, add it (see AnimNotify Setup section)
  2. ‚úÖ Check combo window timing
    • Window should NOT be at 0.0s (animation start)
    • Recommended: 0.3s - 0.8s (middle of animation)
  3. ‚úÖ Verify input action matches
    • Edge TransitionInputAction must match your input
    • Check spelling/capitalization
  4. ‚úÖ Check logs for events
    • Look for: "üü¢ Combo window OPENING"
    • Look for: "‚úÖ Queued transition to 'X'"
    • Look for: "üéØ Executing queued transition"

Combo Loops Infinitely

Common Issue

Symptoms:

  • Second node plays repeatedly without input
  • Combo never ends

Cause:

Old montage callbacks firing after transition (fixed in latest version).

Solution:

  1. Update to latest HopComboGraph version
  2. Rebuild project
  3. Verify OnMontageEnded checks CurrentMontage validity

Animation Doesn't Play

Setup Issue

Checklist:

  • ‚úÖ Montage assigned to node's AnimationMontage property
  • ‚úÖ Character has AnimInstance with correct skeleton
  • ‚úÖ Ability is activated (check GAS logs)
  • ‚úÖ Ability System Component is valid
  • ‚úÖ Character has mesh component

Input Not Detected

Setup Issue

Checklist:

  • ‚úÖ Enhanced Input system enabled
  • ‚úÖ Input Action created and assigned
  • ‚úÖ Input Mapping Context added to player controller
  • ‚úÖ Input Action bound in C++/Blueprint
  • ‚úÖ Ability activated before input pressed

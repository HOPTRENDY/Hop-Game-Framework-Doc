üîó System Integration Summary

Complete Flow

How All Systems Work Together:

1. Input System (HopInputRuntime):
   Player presses attack button
   ‚Üí IA_Attack input action triggered
   ‚Üí HopInputHandler_ComboAttack processes input
   ‚Üí Sends input to HopComboManagerComponent

2. Combo System (HopComboGraphRuntime):
   HopComboManagerComponent receives input
   ‚Üí Checks if combo ability is active
   ‚Üí If yes: buffers input for next state
   ‚Üí If no: activates GA_Sword_LightCombo ability
   ‚Üí Ability starts combo graph (CG_Sword_LightCombo)
   ‚Üí State machine plays montage (Attack 1)
   ‚Üí AnimNotifyState_ComboWindow opens input buffer
   ‚Üí Player presses attack again (buffered)
   ‚Üí State transitions to Attack 2
   ‚Üí Repeat until finisher or cancel

3. Damage System (HopGasBuddy + GAS):
   State enters (Attack 1 starts)
   ‚Üí On State Enter: Apply GE_MeleeDamage_Sword
   ‚Üí Gameplay Effect modifies Health attribute (-25)
   ‚Üí Apply GE_PoiseDamage_Sword
   ‚Üí Poise damage accumulates
   ‚Üí If poise breaks: Add State.Stagger tag
   ‚Üí Target plays hit reaction animation

4. VFX System (Gameplay Cues):
   Montage plays ‚Üí AnimNotify triggers
   ‚Üí Execute GameplayCue.Weapon.Sword.Swing
   ‚Üí GC_Weapon_Sword_Swing spawns trail VFX
   ‚Üí On damage applied: GameplayCue.Impact.Flesh
   ‚Üí GC_Impact_Flesh spawns blood VFX + decal + sound

5. Tag System (All Modules):
   Ability activates ‚Üí Grants State.Combat.Attacking tag
   ‚Üí Blocks sprint, reload, other conflicting abilities
   ‚Üí AI detects State.Melee tag ‚Üí Triggers defensive behavior
   ‚Üí On combo complete ‚Üí Removes all granted tags
   ‚Üí Player can sprint/dodge again

Result: Smooth, responsive melee combat with visual feedback,
network replication, and intelligent AI response - all managed by
interconnected HOP Framework systems using Gameplay Tags as the glue!
